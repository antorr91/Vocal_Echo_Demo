Vocal Echo

A closed-loop system for real-time vocal analysis of domestic chicks.

What is Vocal Echo?

Animal vocalisations are a fundamental way to assess psychophysical states and how environmental conditions influence behavioural responses in early stages of life. Newly hatched chicks (Gallus gallus), as precocial, are a unique model for studying how early vocal interactions affect animal internal states. Understanding these effects requires real-time assessment of calls and precise temporal control of acoustic feedback, challenges that have limited our ability to study vocal interactions in isolated young animals. Here we present Vocal Echo, a closed-loop system that automatically detects vocalisations and provides contingent acoustic feedback, allowing real-time interactions with the chicks. Vocal Echo represents a methodological advance in studying vocal interactions, offering researchers the ability to examine how specific acoustic experiences can influence internal animal states and paves the way for real time investigation of the neural mechanisms underlying vocal communication.

  • Calls Detection: Based on signal processing algorithms, and faster than deep learning methods .
  • Feedback: plays context-specific calls/white-noise/silence within a fixed latency budget.
  • I/O: GUI controls, OSC endpoints.
Get started

Quick start

  1. Install prerequisites:
    • Windows 10/11 (64-bit)
    • Visual Studio 2022 with Desktop development with C++
    • JUCE 8 (includes Projucer)
  2. Create a new JUCE project:
    • Open Projucer → New Project → select GUI Application (⚠ not Audio Plug-In).
    • Set project name VocalEcho and choose Visual Studio 2022 as exporter.
  3. Select modules :
    • juce_audio_basics, juce_audio_devices, juce_audio_formats, juce_audio_utils
    • juce_core, juce_data_structures, juce_dsp, juce_events, juce_graphics, juce_gui_basics, juce_osc

    Do not include juce_audio_plugin_client, juce_product_unlocking, juce_cryptography, juce_video unless required — they are not needed here.

  4. Add source files:
    • Download the project files from GitHub.
    • Copy MainComponent.cpp and MainComponent.h into the project’s Source/ folder.
    • In Main.cpp (generated by Projucer) include your component:
      #include "MainComponent.h" and set it as the main window content.
  5. Build:
    • Save in Projucer → “Save Project and Open in IDE”.
    • In Visual Studio, select configuration Release | x64 and build.
    • The executable will appear under Builds/VisualStudio2022/x64/Release/.
  6. Prepare stimuli:
    • Create a folder calls_fv/ with subfolders: pleasure_condition, contact_condition, cluck_condition.
    • Add files pleasure_A/B/C.wav, contact_A/B/C.wav, cluck_A/B/C.wav.
    • Each folder must also include white_A/B/C.wav and silence_A/B/C.wav.
    • Place beep.wav in the root of calls_fv/.
  7. Run: launch the VocalEcho.exe. Use the GUI buttons or send OSC messages on UDP port 9001 (e.g. /chick/trigger).

Python snippet (OSC control)

from pythonosc.udp_client import SimpleUDPClient c = SimpleUDPClient("127.0.0.1", 9001) c.send_message("/chick/trigger", 1) # 0=silence, 1=biological, 2=white

Source files are available on GitHub. Download and place them in your project’s Source/ directory before building.

Architecture

Signal processing (current)

  • Onset detection: High-Frequency Content (HFC) over a filtered spectrogram with peak picking; global time-shift compensation; refractory logic to avoid double counts.
  • Offset detection: First-order difference on a mel spectrogram within an expected duration window (min/max), picking the steepest negative slope as call.
  • Streaming: Circular audio buffer (real-time mic input), ~10 ms hops for low latency.
  • OSC link: Every detected onset/offset is mirrored via /chick/state; stimulus triggers are sent via /chick/trigger to the JUCE app.

Feature extraction

  • Temporal features: Call duration, Attack time ( envelope derived).
  • Spectral & energy features: Spectral centroid (mean/std), RMS (mean/std).
  • Envelope features: Attack magnitude, Slope (from analytic envelope).
  • Pitch-related features: YIN F0 statistics (mean/std/skew/kurtosis, bandwidth), first-order diff mean, slope; harmonic magnitudes (F1, F2) and ratios (F1/F0, F2/F0).

Planned (not implemented yet)

  • Classifier (Soon available): Label calls (e.g., contact / pleasure / cluck) using a lightweight model.
  • Policy module: Map labels/context to specific feedback strategies.

Latency focus

  • Short frame hops for rapid onset detection; offset estimation bounded by expected durations.
  • Playback in the standalone JUCE app is pre-buffered; triggers are delivered via OSC.

Interface

Main controls

  • Output mode — Mono / Stereo. In stereo, balance is enabled for L/R weighting.
  • Start — initialises session, plays a beep.wav, opens log; press again to stop and save.
  • Biological conditions — selects Pleasure / Contact / Cluck for biological playback.
  • Biological call — plays an A–B–C sequence from the selected family (randomised order), with configured delay and repetitions.
  • White noise — control stimulus; uses the same delay/repetition parameters.
  • Silence — silent control (logs the event, no audio).

Sliders

  • Volume — linear gain applied to playback buffers.
  • Delay (ms) — inter-stimulus interval within the A–B–C sequence (10–2000 ms).
  • Repetitions — number of times the A–B–C sequence is repeated (1–5).
  • Balance — L/R pan when Stereo is active; 0 centred.

Remote control (OSC)

  • /vocal-echo/enable 0/1 — start/stop detection & logging.
  • /vocal-echo/feedback 0/1/2 — silence / biological / white-noise.
  • /vocal-echo/state — telemetry with onset timestamps (read-only).
